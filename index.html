Análisis Técnico: Recorrido Virtual Museo Jumex

Un análisis de consultoría sobre la implementación técnica de la escena 3D.

Veredicto del Estudio

Este es un **proyecto académico robusto y bien estructurado**. La base del código (manejo de shaders, cámara, modelos, skybox) es limpia y sigue las prácticas estándar de la industria para aplicaciones OpenGL.

La implementación del sistema de **iluminación múltiple (Forward Rendering)** es el punto más fuerte, combinando luz direccional, focos estáticos para la galería y un foco dinámico (linterna) adjunto a la cámara. La gestión de recursos (modelos `.obj`) es eficiente para una escena de este tamaño. El código para animación por keyframes y modelos animados (`ModelAnim`) está presente, mostrando una ambición que va más allá de la escena estática.

Conclusión: Un excelente portafolio de habilidades fundamentales en gráficos 3D.

Descripción General del Proyecto

Este es un proyecto de gráficos por computadora escrito en C++ que utiliza OpenGL para renderizar una escena 3D. El objetivo principal es mostrar una recreación del Museo Jumex, incluyendo su arquitectura (dividida en varios pisos) y algunas obras de arte (pinturas) y mobiliario (sillones, extintores) en su interior.

El proyecto implementa una cámara en primera persona (FPS-style) que permite al usuario moverse libremente por la escena. Además, incluye un sistema de animación por Keyframes para crear movimientos predefinidos, como la rotación de todo el museo (controlado por la variable giroMonito).

Tecnologías y Bibliotecas

  • OpenGL (via GLAD)

    Es la API de gráficos principal utilizada para todo el renderizado 3D. glad.h se usa como el cargador de funciones de OpenGL.

  • GLFW

    Se utiliza para crear la ventana de la aplicación, manejar el contexto de OpenGL y procesar las entradas del usuario (teclado y mouse).

  • GLM (OpenGL Mathematics)

    Biblioteca de matemáticas (vectores y matrices) esencial para manejar las transformaciones 3D (mat4, vec3), la proyección y la vista de cámara.

  • stb_image.h

    Biblioteca de un solo encabezado para cargar archivos de imagen (.jpg, .png) y usarlos como texturas en OpenGL.

  • SDL 3

    Se incluye SDL3/SDL.h y se usa específicamente para el control del tiempo (SDL_GetTicks(), SDL_Delay()) para limitar el framerate a 60 FPS.

  • Clases Personalizadas (.h)

    El proyecto está bien estructurado con clases propias como Shader, Camera, Model, ModelAnim, y Skybox para encapsular la funcionalidad.

Análisis del Manejo de Luces

El sistema de iluminación (shader shader_Lights_mod.fs) es un pilar del proyecto. Utiliza un enfoque de **renderizado "forward"** que combina múltiples tipos de luces, calculadas por fragmento.

1. Luz Direccional (Global)

Simula una fuente de luz lejana (Sol). Afecta a toda la escena con intensidad moderada (diffuse: 0.7) y provee la iluminación base.

2. Foco: Linterna (Spotlight[0])

¡Una característica clave! Es un spotLight dinámico. Su posición (camera.Position) y dirección (camera.Front) están ligadas a la cámara, actuando como una linterna para el jugador.

3. Focos: Galería (Spotlight[1-5])

Son 5 focos estáticos. Como dice el código: "LUCES DE FOCO ANARANJADAS".

  • Crean el ambiente de galería.
  • Tienen un color cálido (1.0f, 0.5f, 0.1f).
  • Usan cutOff para crear un haz cónico.
  • Tienen atenuación realista.

4. Luces de Punto (Desactivadas)

El shader soporta 2 pointLight. Sin embargo, en el código C++, ambas (pointLight[0] y pointLight[1]) están efectivamente desactivadas, ya que sus colores están en (0.0, 0.0, 0.0).

Gestión de Modelos 3D y Escena

La escena se construye cargando múltiples modelos 3D estáticos, la mayoría en formato .obj, usando la clase personalizada Model.

Arquitectura (Museo Jumex):

  • vestibulo.obj
  • MuseoJumexTercerPiso.obj
  • MuseoJumexSegundoPiso.obj
  • MuseoJumex.obj

Exhibits y Mobiliario:

  • frame1.obj a frame05.obj
  • sillon.obj
  • extintor.obj

Entorno (Skybox):

Se utiliza la clase Skybox para renderizar un mapa de cubos (cubemap) que simula el cielo y el entorno exterior, dando a la escena una sensación de profundidad y realismo.

Potencial No Utilizado:

El código incluye una clase ModelAnim y una línea comentada para cargar un .dae. Esto sugiere que había planes de incluir modelos animados, pero no están activos en esta versión.

Recomendaciones del Estudio

El proyecto es una demostración sólida. Para llevarlo al siguiente nivel, sugerimos las siguientes áreas de mejora:

  • Activar Iluminación Dinámica

    Activar las pointLight (actualmente desactivadas) y hacer que sigan al jugador o a otros objetos añadiría un gran nivel de dinamismo al interior del museo.

  • Implementar Modelos Animados

    Finalizar la implementación de ModelAnim (archivos .dae o .fbx) permitiría añadir visitantes animados o exhibiciones dinámicas, dando vida a la escena.

  • Explorar Renderizado Diferido (Deferred Shading)

    El renderizado "forward" actual es costoso (calcula N luces por M objetos). Para una escena con muchas luces (como una galería), un sistema de **Renderizado Diferido** sería mucho más eficiente, ya que desvincula el costo de la iluminación del número de objetos.
