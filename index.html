Análisis del Proyecto Gráfico

Un vistazo técnico a la escena 3D del "Museo Jumex".

Descripción General del Proyecto

Este es un proyecto de gráficos por computadora escrito en C++ que utiliza OpenGL para renderizar una escena 3D. El objetivo principal es mostrar una recreación del Museo Jumex, incluyendo su arquitectura (dividida en varios pisos) y algunas obras de arte (pinturas) y mobiliario (sillones, extintores) en su interior.

El proyecto implementa una cámara en primera persona (FPS-style) que permite al usuario moverse libremente por la escena. Además, incluye un sistema de animación por Keyframes para crear movimientos predefinidos en los objetos, como la rotación de todo el museo (controlado por la variable giroMonito).

Tecnologías y Bibliotecas

  • OpenGL (via GLAD)

    Es la API de gráficos principal utilizada para todo el renderizado 3D. glad.h se usa como el cargador de funciones de OpenGL.

  • GLFW

    Se utiliza para crear la ventana de la aplicación, manejar el contexto de OpenGL y procesar las entradas del usuario (teclado y mouse).

  • GLM (OpenGL Mathematics)

    Biblioteca de matemáticas (vectores y matrices) esencial para manejar las transformaciones 3D (mat4, vec3), la proyección (perspective) y la vista de cámara.

  • stb_image.h

    Una popular biblioteca de un solo encabezado para cargar archivos de imagen (como .jpg, .png, TGA) y usarlos como texturas en OpenGL.

  • SDL 3

    Curiosamente, se incluye SDL3/SDL.h. Se usa específicamente para el control del tiempo (SDL_GetTicks(), SDL_Delay()) para limitar el framerate a 60 FPS.

  • Clases Personalizadas (.h)

    El proyecto está bien estructurado con clases propias como Shader, Camera, Model, ModelAnim, y Skybox para encapsular la funcionalidad.

Requisitos del Sistema

Mínimos

  • Procesador: Intel Core i5 (8va Gen) o AMD Ryzen 5
  • RAM: 8 GB
  • Gráficos: Intel UHD 620 o superior (Compatible con OpenGL 3.3+)

Recomendados

  • Procesador: Intel Core i7 (8va Gen+) o AMD Ryzen 7
  • RAM: 16 GB
  • Gráficos: NVIDIA / AMD con 2 GB+ de VRAM dedicada

Interactividad

Controles de Cámara (FPS)

  • Movimiento: Teclas W, A, S, D
  • Vista: Movimiento del Mouse
  • Zoom: Rueda del Mouse (Scroll)

Animación por Keyframes

  • Animación Activa: El museo completo rota sobre su eje (giroMonito).
  • Activación: Tecla 'G' para iniciar/pausar la animación.
  • Reinicio: Tecla 'R' para volver a la posición inicial.

Análisis del Manejo de Luces

El manejo de la iluminación se configura en el shader shader_Lights_mod.fs y se alimenta desde C++ en cada frame. Es un sistema de múltiples luces que combina tres tipos:

1. Luz Direccional

Se define una dirLight global que simula una fuente de luz lejana (como el sol). Tiene componentes ambiente, difuso y especular.

2. Luces de Punto

El shader soporta dos pointLight. Sin embargo, en el código, ambas (pointLight[0] y pointLight[1]) están efectivamente desactivadas, ya que sus componentes de color (ambiente, difuso, especular) están todos en (0.0, 0.0, 0.0).

3. Focos (Spotlights)

Esta es la iluminación principal del interior. Se definen 6 spotLight, pero solo se usan 5 (del 1 al 5).

Detalle Clave de los Focos:

El comentario en el código lo explica perfectamente: "LUCES DE FOCO ANARANJADAS DE ALTA INTENSIDAD PARA CADA CUADRO".

  • Cada foco (del 1 al 5) apunta a una de las pinturas.
  • Tienen un color naranja/cálido (ej. diffuse: (1.0f, 0.5f, 0.1f)) para dar ambiente de galería.
  • Usan cutOff y outerCutOff para crear un haz de luz cónico y enfocado.
  • Tienen atenuación (constant, linear, quadratic) para que la luz pierda intensidad con la distancia.

Gestión de Modelos 3D y Escena

La escena se construye cargando múltiples modelos 3D estáticos, la mayoría en formato .obj. Son cargados usando la clase personalizada Model.

Arquitectura (Museo Jumex):

  • vestibulo.obj
  • MuseoJumexTercerPiso.obj (Cargado como primerPiso)
  • MuseoJumexSegundoPiso.obj (Cargado como segundoPiso)
  • MuseoJumex.obj (Cargado como tercerPiso)

Exhibits y Mobiliario:

  • frame1.obj (Pintura)
  • frame02.obj (Pintura)
  • frame03.obj (Pintura)
  • frame04.obj (Pintura)
  • frame05.obj (Pintura)
  • sillon.obj
  • extintor.obj

Entorno (Skybox):

Se utiliza la clase Skybox para renderizar un mapa de cubos (cubemap) que simula el cielo y el entorno exterior, dando a la escena una sensación de profundidad y realismo.

Modelos No Utilizados:

El código incluye una clase ModelAnim y una línea comentada para cargar un archivo .dae (//ModelAnim animacionPersonaje...). Esto sugiere que había planes de incluir modelos animados, pero no están activos en esta versión del código.

Conclusión y Posibles Mejoras

El proyecto es una demostración sólida de los fundamentos de OpenGL. Logra construir una escena compleja a partir de múltiples modelos, implementa un sistema de iluminación múltiple efectivo (especialmente los focos) y una cámara FPS funcional.

Áreas de Mejora:

  • Optimización de Luces

    Las luces de punto (pointLight) están definidas en el shader pero desactivadas en C++. Se podrían activar para iluminar dinámicamente el interior a medida que el jugador se mueve.

  • Animación de Modelos

    Implementar la carga de ModelAnim (archivos .dae) permitiría añadir personajes animados o exhibiciones dinámicas, llevando el proyecto al siguiente nivel.

  • Renderizado Diferido (Deferred Shading)

    Para una escena con muchas luces (como una galería), un sistema de renderizado diferido sería mucho más eficiente que el renderizado "forward" actual, permitiendo cientos de luces con bajo costo de rendimiento.
